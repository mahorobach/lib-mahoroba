# 本棚デジタル化プロジェクト 引き継ぎノート
作成日：2026-06-05

---

## プロジェクトの発端

「本棚の本を処分したい」→「処分前にデータとして残したい」→「要約を音声で聴けるアプリを作りたい」という流れで発生。

オーディブルの代替アプリ（個人用）を作ることが技術的な目標。

---

## 今回の対話で明確になったこと

### アプリの要件

**目的：** 本の内容を、必要な時に音声で呼び出せる個人ライブラリ

**2つのモード：**
- 【要約モード】作業中・移動中に聴き流す
- 【原文モード】キーワードで検索・参照する

**技術スタック案：**
| 要素 | 手段 |
|---|---|
| フロントエンド | HTML/CSS/JavaScript |
| 要約生成 | Claude API |
| 音声合成 | Web Speech API（ブラウザ内蔵・無料） |
| データ保存 | localStorage or Cloudflare KV |
| ホスティング | Cloudflare Pages |

**データ構造：**
```json
{
  "title": "書名",
  "category": "カテゴリ",
  "summary": "要約テキスト",
  "full_text": "原文テキスト",
  "tags": ["タグ1", "タグ2"]
}
```

## 本棚の仕分け結果

### 仕分け軸
「この本は自分の根拠に関わるか」

### カテゴリ別判断

| カテゴリ | 判断 | 理由 |
|---|---|---|
| 日本史・古典 | 残す or 要約 | Mahoroba Channel・道徳会館と直結 |
| 古事記・ほつまつたゑ | 要約＋保管 | 長期資産 |
| 南総里見八犬伝 | 青空文庫で代替 | 物理本は処分候補 |
| SwiftUI等IT技術書 | 内容確認後判断 | 陳腐化している可能性あり |
| ビジネス・自己啓発 | 原則処分 | 内容を覚えていない＝接続していない |
| **フォースかパワーか** | **保管・構造化** | **判断軸として機能している** |
| **言霊百霊** | **保管・構造化** | **Mahoroba Channelの原動力** |

### 処分候補筆頭
- DaiGo系・一般ビジネス書
- 時事性の高い金融書

## 思想的な核（重要メモ）

今回の対話で浮上した構造：

```
言霊百霊（言葉＝神という認識）
　↓
古事記の新しい読み方
　↓
まほろばチャンネルの原動力
　↓
道徳会館での講義・MC
```

**フォースかパワーか** と **言霊百霊** は単なる本ではなく、活動全体の判断軸・根拠になっている。この2冊はObsidianで別途構造化ノートを作る価値がある。

---

## ユーザータイプの発見（将来の応用メモ）

本棚仕分け作業には2タイプのユーザーがいる：

- **タイプA：** 目的明確（断捨離・売却）→ 深掘り不要、基準を渡して作業させる
- **タイプB：** 本に思想・歴史が詰まっている → 深掘りが必要、判断軸の言語化が先

→ アプリ設計またはコンテンツ化の際に参照すること。ただし今は実装を優先。
