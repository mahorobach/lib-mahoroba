# 個人蔵書音声ライブラリ アプリ

## プロジェクト概要

個人の蔵書を要約・音声化して、必要な時に呼び出せるWebアプリ。
オーディブルの個人用代替ツール。

## このアプリの本質

**「まほろばチャンネルの源泉となった本棚を、声で呼び出せるアプリ」**

この本棚はコレクションではなく、Mahoroba Channelのアイデアと思想の源泉として集められたものだ。
本の知恵を必要な時に声として呼び出すことが、このアプリの存在理由。

## アプリ名

暫定：未定（まほろばライブラリーは自分用としては適切）
公開・配布時は別途検討。本質は「源泉を声で呼び出す」こと。

## 機能要件

### MVP（最初に作るもの）

1. **テキスト入力** - 本のテキストをペーストまたはファイルアップロード
2. **要約生成** - Claude APIで章ごと・全体要約を生成
3. **音声再生** - Web Speech APIで要約テキストを読み上げ
4. **ライブラリ管理** - タイトル・カテゴリ・タグで管理

### 2モード設計

```
【要約モード】
- 短い要約テキストを音声再生
- 作業中・移動中の聴き流し用
- 再生速度調整可能

【原文モード】
- キーワード検索
- 該当箇所をテキスト表示
- 必要に応じて音声再生
```

### 将来機能（MVPには含めない）
- OCRによるスキャン本の取り込み
- 複数デバイス同期
- 音声品質向上（OpenAI TTS等への切り替え）

## 技術スタック

| 要素 | 技術 | 備考 |
|---|---|---|
| フロントエンド | HTML/CSS/JavaScript | シンプルに保つ |
| 要約生成 | Gemini API (gemini-2.5-flash-lite) | |
| 音声合成 | Web Speech API | ブラウザ内蔵・無料 |
| データ保存 | localStorage（MVP） → Cloudflare KV | |
| ホスティング | Cloudflare Pages | qualifications.appと同環境 |

## データ構造

```json
{
  "id": "uuid",
  "title": "書名",
  "author": "著者",
  "category": "カテゴリ",
  "tags": ["タグ1", "タグ2"],
  "summary_short": "3行要約",
  "summary_full": "章ごと要約",
  "full_text": "原文テキスト（任意）",
  "created_at": "ISO8601",
  "source": "青空文庫 | 自炊 | 手入力"
}
```

## カテゴリ定義

- `思想・判断軸` - フォースかパワーか等、生き方の根拠になる本
- `日本史・古典` - 古事記、ほつまつたゑ、里見八犬伝等
- `技術・IT` - Swift、Web技術等
- `ビジネス` - 経営・金融等
- `その他`

## 開発方針

- **まずWebアプリとして完成させる**（iOS化は後回し）
- **テストデータは青空文庫の古事記から始める**
- **音声品質よりも動くことを優先する**（Web Speech APIで十分）
- シンプルに保つ。機能追加は使いながら判断する

## 注意事項

- 著作権のある本のテキストは個人利用の範囲内で扱うこと
- OCR精度は図表の多い本で落ちる（金融書等は手動補正が必要）
- Web Speech APIの音声品質は日本語でブラウザによって差がある
